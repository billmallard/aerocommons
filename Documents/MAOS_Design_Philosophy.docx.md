**MOBILE AVIATION OPERATING SYSTEM**

*Project Design Philosophy & North Star Statement*

Version 0.1  //  March 2026  //  DRAFT

# **North Star Statement**

*This aircraft is designed as a complete mobile aviation operating system. The airplane and trailer are co-equal components, designed together from first principles. The system requires no fixed infrastructure — no hangar, no FBO, no ground crew. A single pilot can deploy, operate, and recover the system from any suitable airstrip.*

*Every design decision is evaluated against this principle. Complexity that serves the operating system is acceptable. Complexity that serves only aesthetics or marginal performance gains is rejected. The airplane must be buildable, maintainable, and operable by one person with the tools that travel with it.*

This is the filter through which every design decision passes. When tempted to add a feature, add weight, or add complexity, return here first.

# **What This Is — And Is Not**

This is not a trailerable airplane. Trailerable airplanes tolerate being trailered. This system is designed around the trailer as a first-class component from day one.

The distinction matters because it changes every priority:

* A trailerable airplane compromises to fit on a trailer. This airplane is designed with the trailer as its ground station.

* A trailerable airplane stores in a hangar and occasionally gets trailered. This system has no hangar — the trailer is the hangar.

* A trailerable airplane requires two people and a forklift to rig. This system deploys solo, in under an hour, from a pickup truck.

The closest analogy is a boat on a trailer. The owner arrives at any suitable body of water, launches, operates, recovers, and drives away. No marina slip required. No dock fees. No dependence on fixed infrastructure. This is that — for aviation.

# **What This System Enables**

The operating system concept transforms the owner's cost structure and operational freedom completely:

* No hangar lease — potentially the single largest recurring cost in personal aviation, eliminated.

* No dependence on FBO services for routine operations.

* Maintenance at home, in the trailer, with the integrated workbench and tool storage.

* The airplane can be stored at home, in a driveway, legally — like a boat.

* Fly from any suitable strip in a region, not just home base. Tow to a new area and operate from there.

* True cross-country independence: depart from a grass strip, land at a paved field, drive the trailer to meet yourself if needed.

The lifestyle this enables is different in kind, not just degree. This is not flying from Airport A to Airport B and back. This is towing your operating system to a region, flying from multiple local strips, and moving on when ready.

# **Design Philosophy**

## **Rutan's Rule**

Safety through inherent stability. The configuration must resist departure without demanding pilot heroics. A bad moment must not become a fatal moment.

## **Bede's Rule**

If a determined amateur with modest tools cannot build this, the design has failed before it flew. Complexity is a bug, not a feature. A sophisticated airplane can and should be buildable by one person in a two-car garage on a reasonable budget.

## **The Operating System Rule**

Every component — airframe, powerplant, systems, trailer, crane — is evaluated as part of a single integrated system. A component that is excellent in isolation but degrades the operating system is the wrong component.

When a design decision is genuinely uncertain, ask: which choice better serves the solo operator at a remote strip with the tools in the trailer?

# **Design Priorities**

In strict order. A lower priority never overrides a higher one.

1. Stall/departure resistance — canard or inherently safe configuration preferred.

2. Structural simplicity — minimum part count, minimum joints, minimum skill at each build step.

3. Repairability in the field with basic tools and materials from Aircraft Spruce or equivalent.

4. Performance — efficiency and speed matter, but not at the cost of priorities 1–3.

5. Aesthetics — a good design looks right, but ugly flies fine.

# **The Decision Filter**

Before accepting any design feature, ask these questions in order:

* Does this serve the operating system, or only the airplane?

* Can a competent amateur build or repair this with tools that travel in the trailer?

* What does this weigh, and does it justify that weight?

* If this fails at a remote strip, what is the recovery path?

* Does "that's how production aircraft do it" justify this — or is that a lazy answer?

If a feature cannot survive this filter, it does not belong in the design. Note the idea in the design log and move on.

# **Non-Negotiable Constraints**

These are not preferences. They are constraints that define what this project is.

* Never propose a feature that requires a professional machine shop to fabricate.

* Never accept retractable gear on the primary design — the complexity and failure modes are incompatible with the operating system concept.

* Never optimize for aesthetics at the expense of buildability.

* Never ignore weight — always ask what the lightest acceptable solution is before proposing a heavier one.

* Never design around a single-source supplier with no track record in the homebuilder community.

# **Current Configuration Snapshot**

This is the working design as of initial concept phase. All figures are targets subject to revision.

## **Airframe**

**Configuration:**  High-wing, pod-and-boom, tricycle gear, fixed faired main gear

**Seats:**  4 (wide cabin — 46" minimum, Cherokee Six reference)

**Wing:**  One-piece, removable, tubular aluminum spar (BD-4 inspired), laser-cut ribs

**Wing area:**  \~140 ft²  |  Span: \~37.4 ft  |  AR: 10  |  Loading: \~17.5 lbs/ft²

**Airfoil:**  Riblett GA35-615 (15% root / 12% tip) — working assumption

**Primary structure:**  Aluminum semi-monocoque; flat-cut parts from SendCutSend

**IFR / Ice:**  IFR capable; TKS ice protection preferred

## **Powerplant (Primary Path)**

**Engine:**  Viking 195T (Honda-based, 195 HP, turbocharged, FADEC, mogas)

**Alternates:**  ULPower UL520T; Kawasaki Inline 4 Turbo (provision only, pending certification)

**Cooling:**  Liquid-cooled; heat exchanger provides cabin heat

## **Performance Targets**

**Cruise:**  155 KTAS

**Range:**  1,000 nm  |  Endurance: 7 hrs (5 hrs flying \+ reserves)

**Useful load:**  1,200 lbs including fuel

**Gross weight:**  \~2,430 lbs

**Empty weight target:**  \~1,230 lbs

**Service ceiling:**  17,500–20,000 ft

## **Trailer System**

**Concept:**  Integrated ground station — horizontal boom crane deploys from trailer roof

**Crane:**  Horizontal boom, min. half-wingspan reach; rotating cradle engages wing at strut fittings

**Sequence:**  Disconnect 4 pins \+ QD fittings, lift 6", rotate 90°, retract — wing stowed

**Target time:**  \~65 min solo (45 min with practice)

**Trailer features:**  Shore power, inverter, workbench, tool storage, wing stowed above fuselage

**Regulatory:**  FAA Experimental Amateur-Built (51% rule)
